## Tailwind v4 Rules
- Tailwind v4 via `@tailwindcss/postcss`. CSS-first config — there is no `tailwind.config.js/ts`.
- CSS variables live in `src/app/globals.css` under `@theme inline { ... }`.
- Always use semantic design tokens: `bg-background`, `text-foreground`, `border-border`, `text-muted-foreground`, `bg-primary`, `text-primary-foreground`, `bg-muted`, `bg-card`, etc. Avoid raw color classes like `bg-gray-500`.
- Custom classes go in `globals.css` inside `@layer utilities` or `@layer components`. Never create separate CSS files.
- Use `tw-animate-css` utilities for animations (`animate-in`, `fade-in`, `slide-in-from-top-1`, `zoom-in-95`, etc.).
- Always merge classes with `cn()` from `@/lib/utils` (wraps `clsx` + `tailwind-merge`).

## Component Authoring Rules
- **TypeScript strict**: no `any`, ever. Define props as explicit interfaces in the component file. If a type is shared across files, place it in `src/types/<name>.md.ts` (preserve the `.md.ts` suffix — it is intentional).
- **`"use client"`** only when necessary: event handlers, `useState`, `useEffect`, refs, browser APIs, or client-only libraries. Default to React Server Components.
- **Forms**: always `react-hook-form` + `@hookform/resolvers/zod` + `zod` v4. Never uncontrolled inputs for form state. Validation schemas for shared forms go in `src/validations/<name>.md.ts`.
- **Toasts**: import `toast` from `"sonner"`. The `<Toaster />` is mounted via providers — do not remount it.
- **Dialogs/Sheets**: shadcn `Dialog`, `AlertDialog`, `Sheet`, `Drawer`. Never native `<dialog>`.
- **Data tables**: use shadcn `Table` primitives (`Table`, `TableHeader`, `TableBody`, `TableRow`, `TableHead`, `TableCell`).
- **No comments** unless the WHY is non-obvious. Never narrate what code does.
